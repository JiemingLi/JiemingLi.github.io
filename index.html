java反射理解

发表于 2019-03-18 | 分类于 Java基础

什么是反射

​ 反射就是指程序在运行的时候可以知道一个类的自身信息。

​ 对于任何一个类：可以知道这个类的属性和方法。

​ 对于任何一个对象：可以调用这个对象的任何一个方法和属性。

​ 反射就是把java类中的各种成分映射成一个个的Java对象

​ 例如：一个类有：成员变量、方法、构造方法、包等等信息，利用反射技术可以对一个类进行 解剖，把个个 组成部分映射成一个个对象。

​ （其实：一个类中这些成员方法、构造方法、在加入类中都有一个类来描述）

反射的功能

  • 在运行时判断任意一个对象所属的类
  • 在运行的时候构造任意一个类的对象
  • 在运行时判断一个类所具有的成员变量和方法
  • 在运行时调用任何一个对象的方法
  • 生成动态代理（会有一篇关于动态代理的文章，在这里挖坑）

反射的优点和缺点

优点

  • 以实现动态创建对象和编译，体现出很大的灵活性，特别是在J2EE的开发中它的灵活性就表现的十分明显。比如，一个大型的软件，不可能一次就把把它设计的很完美，当这个程序编译后，发布了，当发现需要更新某些功能时，我们不可能要用户把以前的卸载，再重新安装新的版本，假如这样的话，这个软件肯定是没有多少人用的。采用静态的话，需要把整个程序重新编译一次才可以实现功能的更新，而采用反射机制的话，它就可以不用卸载，只需要在运行时才动态的创建和编译，就可以实现该功能。

缺点

  • 对性能有影响。使用反射基本上是一种解释操作，我们可以告诉JVM，我们希望做什么并且它满足我们的要求。这类操作总是慢于只直接执行相同的操作。

工厂模式

  1. 内容

    把工厂创建对象的行为抽象成接口或者抽象类（工厂接口或者抽象工厂类），让它的实现类或者子类来决定创建什么对象。

  2. 例子

    比如每个手机厂商都具有生产手机的行为，把手机工厂抽象起来成为一个接口或者抽象类，把生产手机的行为抽象为其子类或者实现类必须执行的方法。根据Java的多态，这样调用者不用知道具体是哪个厂商生产，只要保持接口不变，调用者根据接口或者抽象类去调用对应的方法就好了。

  3. 角色

    抽象工厂

    具体工厂 （安卓，苹果）

    抽象产品 （手机）

    具体产品 （安卓手机，苹果手机）

    每一个具体的产品对应具体的工厂。

  4. 优点

    每一个产品对应一个工厂，不需要修改工厂类的代码，换句话说，

    安卓手机给安卓厂商自己做，不需要零件商做。

  5. 缺点

    每一个对象创建就需要一个工厂，当程序中需要很多对象的时候，则会创建很多工厂。

抽象工厂

  1. 内容

    在工厂模式中，每个产品需要相对应的工厂来生产，工厂方法具有唯一性。假如我们希望一个工厂可以创建多个对象，不是单一的某一个对象，这些对象之间也互相有联系。比如说小米手机公司，不单生产手机，还根据制造出手机壳等等。

  2. 角色

    抽象工厂角色（生产一套产品的抽象手机厂商）

    具体抽象工厂角色 （小米手机厂商）

    抽象产品角色

    具体产品角色

    客户端

    对比工厂模式，抽象工厂模式主要在于生产一套产品，不是单个产品，所以抽象工厂的时候，把几个互相有关系的对象放在抽象类或者接口里面即可。

  3. 概念

    • 产品结构等级：产品等级结构即产品的继承结构，比如一个抽象类是手机，其子类可以是iphone，华为手机，小米手机等。则手机与具体的品牌手机之间构成了一个产品等级结构。
    • 产品族：表示指由同一个工厂生成，处于不同产品结构等级的一组产品。比如小米公司生产的小米手机壳处于手机壳结构等级。小米公司生产的手机处于手机结构等级。则小米手机和小米手机壳构成了一个产品族。
    • 抽象工厂模式所提供的一系列产品就组成一个产品族；而工厂方法提供的一系列产品称为一个等级结构。

总结

  • 抽象工厂模式是工厂方法模式的升级版本，他用来创建一组相关或者相互依赖的对象。他与工厂方法模式的区别就在于，工厂方法模式针对的是一个产品等级结构；而抽象工厂模式则是针对的多个产品等级结构。
  • 无论是简单工厂模式，工厂方法模式，还是抽象工厂模式，他们都属于工厂模式，在形式和特点上也是极为相似的，他们的最终目的都是为了解耦。

SringAOP的动态代理方式

何为代理？
  1. 官方说法：给某一个对象提供一个代理对象，并且由代理对象控制目标对象
  2. 举个例子：每个明星都有自己的经纪人。（明星是真是执行对象，经纪人是代理对象）假如你是某地产商，也许会需要请明星来为你的开业活动助兴，也就是说请明星来唱歌跳舞。假如我们直接请明星，是会失败的(因为明星这么忙，没有时间谈这种琐事，会和你说你找我的经纪人讨论这件事请吧)，则我们直接找到经纪人，如果顺利的话，则可以请到这位明星，而经纪人的存在价值，就是拦截你直接去找明星。（代理对象的存在价值就是拦截客户直接访问真实的对象）但是，我们怎么知道明星会做些什么呢，所以我们就需要通过询问代理人明星会做什么，比如唱什么类型的歌，表演什么样类型的节目。因为只有代理人才知道明星会做什么，也就是说，真实执行方法的对象的方法要和代理对象的方法是要一致的
在Spring中为什么要代理？

​ 在应用程序的业务方法中，有些辅助功能（安全校验，事务）是很多业务方法都需要执行的，而这些辅助功能不是业务方法应该关注的点。假如把辅助功能写入每一个业务方法里，第一增大了代码的耦合性，假如在很多方法执行了一样的代码，以后需要修改某个功能将是个噩梦。第二是这些辅助功能与业务方法毫无逻辑关系，是业务方法应该关注的点，会不利于开发人员的开发。

计算机网络--传输层

发表于 2019-03-16 | 分类于 计算机网络

传输层

  • 协议

    1. TCP 传输控制协议
      • 将需要传输的数据进行分段并编号。
      • 建立会话 实现可靠传输，当有部分数据没有到达时，会通知发送方继续传输丢失的部分数据。
      • 流量控制 流量不匹配时，会通知发送数据方控制速度。
    2. UDP 用户数据报协议
      • 一个数据包就可以实现传输
      • 不分段
      • 不建立会话，所以不是可靠传输
      • 不需要流量控制

  • 服务和应用层协议的关系

    • 服务使用TCP或者UDP的端口侦听客户端的请求
    • 客户端使用IP地址定位服务器的位置，端口定位服务
    • 可以在服务器的网卡设定特定的端口开放，实现服务器网络安全

  • 安全

    • 查看端口开放的情况 netstat -n, netstat-a ,netstat -an ,netstat -nb
    • 测试端口是否开放和连接 telnet ip port
    • 回环地址 127.0.0.1 不经过网卡
    • 在网卡设置特定的端口开放

  • 邮件发送的过程

    • 用户将写好的邮件发送给邮件服务器，用的是SMTP协议，则服务器要有SMTP的服务和开放25端口。也就是写好的邮件发给 邮件服务器Ip:25。
    • 邮件服务器收到邮件之后，则把数据放到缓存队列当中，然后运行SMTP协议的进程的邮件服务器和接受端的邮件服务器建立TCP连接，然后发送邮件
    • 当邮件到达对方的邮件服务器的时候，用户查看收件箱，则会用POP3协议，也就是向接收端邮件服务器110端口访问，则会返回邮件。

端口 16位来表示

  1. 熟知端口 0-1023 计算机本来就有的端口
  2. 登录端端口 1024 - 49151 服务开发商在组织注册的端口
  3. 客户端端口49152 - 65535

segment （分段）: 传输层

package （封装成包） : 网络层

fragment （分片）: 数据链路层

UDP

  1. 面向无连接服务，发送数据之前不需要建立连接

  2. 尽最大努力交付，一个数据包就完事，不保证可靠传输，也不保证拥塞控制

  3. 面向报文，适合多媒体传输

  4. 一对多，一对一，多对多通信方式

  5. 首部开销只有8个字节

  6. UDP首部的检验和的值是计算伪首部的长度（网络层的一些信息）加上UDP数据包的长度之和

TCP

  1. 面向连接得服务
  2. 全双工通信
  3. 面向字节流传输数据 ：把文件按照字节，随机传输到TCP缓存区，然后又随机地按照字节进行封装，传输到接收方的TCP缓存接收区，接收方的缓存区再随机地将字节传输到上层，按字节组成文件
  4. 每一条TCP连接只有两个端点，只能是点到点通信 ：套接字表示点（IP+PORT）
  5. 提供可靠交付通信

TCP实现可靠传输

  1. ARQ协议
  2. 停等
  3. 滑动窗口发送 固定的滑动窗口，发送滑动窗口内的字节，收到滑动窗口第一个字节的确认之后，往后移，缓存区掉字节，然后再把滑动窗口内未发送的字节发送出去
  4. 累计确认 连续发送几个字节的数据，如果前面无差错，则发送最后一个字节的确认，如果有差错，则发送最后一个连续的最后一个字节的确认

TCP实现流量控制

  1. 不断地调整接收端的接收窗口，从而调整发送端的发送窗口，进行流量的控制。

三次握手

  1. 有syn，ack，seq，并且告诉对方能接收的最长字节（mms）和接受缓存区的大小。
  2. 当一方收到另外一方的数据之后，根据发过来的接受缓冲区的大小设置发送窗口的大小，值和收到的接受缓冲区大小的值一样

拥塞避免

  1. 发送窗口刚开始由拥塞窗口控制，到后面由接受窗口控制 发送窗口 = min(rwind,gwind)，通过拥塞窗口去设置发送窗口。
  2. 快重传 当接受端发现有数据缺失的时候，连续发送三个确认信号，即表示当前网络快要发生拥塞，让发送窗口做出调整，更好地实现拥塞避免。

TCP建立连接

  1. 三次握手
      • 对于客户端来说，为了证明到达服务端是畅通的，所以要得到确认才可以认为是客户端到服务端是通的，而这个确认就是从服务端发送来。
      • 对于服务端来说，为了证明到客户端是畅通的，所以要得到确认才可以认为是客户端到服务端是通的，这个确认是由客户端发送来。
    2. 三次握手的本质就是 我发信息给你，你让我知道你收得到，你发信息给我，我发信息给你让你知道我收得到。
  2. 参考：https://www.cnblogs.com/zhuzhenwei918/p/7465467.html，https://blog.csdn.net/lengxiao1993/article/details/82771768

TCP 释放连接

  • 四次挥手
    • 客户端：需要释放连接 FIN
    • 服务端：我收到你的要求了(ACK)，不过得等下，你先别说话（客户端不可以向服务端发送数据），让我把剩下的话说完(传输未传输完成的数据)
    • 服务端：好的，我说完了，我可以不说了 FIN
    • 客户端：好,我收到信息了（ACK）
    • 客户端等待2MSL，确认服务端收到信息，如果服务端收不到信息，则会重发数据，这样客户端也可以及时做出响应。

计算机网络--网络层

发表于 2019-02-10 | 分类于 计算机网络

服务

  1. 选择路径，尽可能地转发数据包，不负责数据包的丢失和接收顺序的处理，该功能由传输层实现。

协议

  1. Arp 将目标IP地址广播，目标MAC地址是ff-ff-ff-ff-ff-ff(广播MAC地址)，扫描本网段的MAC地址，得到最终目标IP地址所对应的MAC地址

  2. 静态修改IP地址对应的MAC地址 arp -s (static) 目标IP 最终修改的MAC

    ICMP

    • ping 命令就是实现ICMP协议
    • ping ip -t 不断ping目标IP
    • ping ip -i size设置TTL为size,可以跟踪数据报途径的路由器
    • 目标不可到达是数据帧没有到达目标主机的路径，路由器会发一个ICMP的数据给源主机
    • 请求超时是数据帧到达目标主机了，但是目标主机传回不了给源主机，源主机一直收不到回应，则超时
    • pathping :追踪数据报的路径，并计算丢包情况
    • tracert ip :跟踪数据的转发路径

    IGMP

    • 点到点：可以建立会话，全双工通信
    • 广播
    • 多播：计算机的网卡可以绑定多播地址，不同的多播地址可以收到不同的数据帧，可以跨网段多播。优点：节省带宽。如果某个网段没有绑定对应的多播地址，则此网段将不会收到多播的数据。

    IP

    • IP地址的首部固定长度为20个字节
    • IP地址的总长度最大为65535个字节
    • 数据帧的数据部分最短为46个字节，最长为1500个字节
    • 数据帧的数据部分的长度 = IP首部（20个字节） + 数据段的长度（TCP/UDP首部+数据长度）<=1500bytes
    • 如果IP一层的数据包长度大于1500个字节，则需要在数据链路层进行分片，然后加上IP地址，再进行帧封装
    • 接受端的网络层如果收到分片的数据帧上传的数据包，则进行组合，再传给传输层。
    • 首部的标识是指数据包的编号（传输层进行分段，网络层封装成数据包），标记 是如果数据包分片，则表是是第几片，是不是最后一片等信息。

    • MTU:最大传输单元属于 IP层 <=1500 ， MSS(不包括TCP头部):maxnum segment size 属于传输层 mss <= 1460

    • 如果传输层的MSS > 1460 也就是说在MTU会大于1500，则会在网络层进行分片。

    • 但是分片会花费主机，路由器更多的时间和CPU资源去处理分片，比如分片重组等等。 也可以不进行分片，

    • 如果MSS>1460则丢掉，然后返回主机一个ICMP数据包。

    • （例子）ICMP协议里有type字段，还有code字段，发送type=3，code=4，MTU=1000的消息就可以了
